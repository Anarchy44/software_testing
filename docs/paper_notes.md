# Paper Notes

## 1. FluxEV (WSDM 2021)

**论文**: *FluxEV: A Fast and Effective Unsupervised Framework for Time-Series Anomaly Detection*
**作者**: Jia Li, Shimin Di, Yanyan Shen, Lei Chen

### 问题动机

大型互联网公司需实时监控数百万条 KPI 时序曲线。现有方法痛点:
- **SPOT/DSPOT**: 仅对分布中的极端值敏感, 实际异常往往是"非极端波动"
- **SR-CNN(有监督)**: 需要大量标注, 不可行
- **Donut(VAE)**: 训练耗时长, 需 GPU, 冷启动困难, 数据漂移后需重新训练

核心洞察: 将"非极端的季节性波动"转化为"极端值", 再利用 EVT(极值理论)自动设定阈值。

### 四大组件

```
[数据预处理] → [波动提取] → [两步平滑] → [自动阈值(SPOT+MOM)]
```

#### (1) 数据预处理 (Sec 4.1)
- 短缺失段(<5点): 一阶线性插值
- 长缺失段(≥5点): 周期填充 = X_{t-l} + (μ_t - μ_{t-1})/2

#### (2) 波动提取 (Sec 4.2)
- EWMA 预测器: `E_i = X_i - EWMA(X_{i-s, i-1})`
- 参数 s=10(窗口), α=衰减因子

#### (3) 两步平滑 (Sec 4.3) — 核心创新
- **第一步(Δσ平滑)**: `F_i = max(σ(E_{i-s,i}) - σ(E_{i-s,i-1}), 0)`
  只有导致局部标准差增加的波动才保留
- **第二步(周期max差分)**: `S_i = max(F_i - max(M_{i-l}, M_{i-2l}, ...), 0)`
  只有超过历史同期最大值的波动才认定为异常
- 参数 d=2(漂移容忍半窗口), p=5(回溯周期数)
- 消融实验: 两步平滑将 F1 从 0.342 → 0.666, 提升 94.7%

#### (4) 自动阈值 (Sec 4.4)
- SPOT(基于EVT第二定理): 建模尾部GPD分布
- MOM(矩估计法)替代MLE: 速度提升4-6倍, 精度几乎不变
- 公式: `th_F = t + (σ/γ) * ((q·n/N_t)^{-γ} - 1)`
- 初始阈值 t = 98%分位数

### 实验结果

| Dataset | Algorithm | F1-Score |
|---------|-----------|----------|
| KPI (592万点) | **FluxEV** | **0.790** |
| KPI | SR-CNN (有监督) | 0.771 |
| KPI | Donut | 0.729 |
| Yahoo (57万点) | **FluxEV** | **0.666** |
| Yahoo | Donut | 0.058 |

### 选为复现目标的理由
- 无监督: 不需要标注数据
- 快速: 仅需2个周期+1000点冷启动, MOM加速比4-6x
- 轻量: 代码量小, 无需GPU, 流式可部署
- SOTA: KPI上无监督最优(0.790), 超越有监督SR-CNN

## 2. Donut (WWW 2018)

**论文**: *Unsupervised Anomaly Detection via Variational Auto-Encoder for Seasonal KPIs in Web Applications*
**作者**: Haowen Xu, Wenxiao Chen 等 (清华 + 百度 + 腾讯)

### 问题动机
与 FluxEV 相同: 大规模KPI异常检测, 无标注。但采用深度生成模型路线。

### 三大创新

#### (1) M-ELBO (修改版证据下界)
- 标准VAE训练排除含异常窗口 → 性能差
- Donut反其道: 保留异常窗口, 但掩码异常点: `α_w=1(normal), 0(anomaly)`
- 必须有异常数据训练, 否则VAE学不会为异常输入找后验

#### (2) 缺失数据注入
- 训练时随机将正常点设为0(模拟缺失)
- 数据增强, 放大M-ELBO效果

#### (3) MCMC 插值
- 检测时对缺失点MCMC迭代填充(M=10次): 采样z→重构x_m→更新x_m
- 减少缺失值对潜在表示的偏差

#### KDE 理论解释 (Sec 5)
首次从理论上解释VAE重建概率为何有效:
- 正常窗口q(z|x)形成"时间梯度"(相邻时间→相邻z区域)
- 重建概率等价于加权核密度估计

### 实验结果

| Dataset | Donut (0%标注) | Opprentice (有监督) |
|---------|---------------|-------------------|
| KPI-A (平滑) | F≈0.90 | F≈0.85 |
| KPI-B (中等) | F≈0.85 | F≈0.80 |
| KPI-C (噪声) | F≈0.75 | F≈0.75 |

无监督Donut全面超越有监督Opprentice。

### 不选为复现目标的理由
- 需要GPU训练: 不符合项目基础设施
- 冷启动成本高: 需要完整历史数据训练(本实验仅720点合成数据)
- 通用性差: Yahoo F1仅0.058 (FluxEV 0.666)
- 不适合小数据集: VAE需要大量训练样本

## 3. 两篇论文对比

| 维度 | FluxEV | Donut |
|------|--------|-------|
| **方法** | 统计 + 极值理论 | 深度生成模型 (VAE) |
| **特征** | EWMA + 两步平滑 | VAE 潜在表示学习 |
| **周期性处理** | 显式周期max差分 | 隐式(滑动窗口+时间梯度) |
| **阈值** | SPOT+MOM 自动 | 手工设定 |
| **理论基础** | 极值理论(EVT) | KDE解释(VAE重建) |
| **效率** | 极快(589s/KPI) | 慢(GPU训练+MCMC) |
| **冷启动** | 2周期+1000点 | 完整历史数据 |
| **KPI F1** | 0.790 | 0.729 |
| **Yahoo F1** | 0.666 | 0.058 |
| **部署难度** | 低(无GPU/无训练) | 高(需训练/调参) |

## 4. 本项目策略

选择 FluxEV 为主复现目标, Donut 为对比阅读:
- 实现 FluxEV 完整四大组件(数据预处理→EWMA→两步平滑→SPOT+MOM)
- 在合成KPI数据上验证(F1=0.933) + 在真实故障注入数据上验证
- 进行消融实验(无平滑→仅Δσ→两步平滑), 验证论文核心贡献
- 报告中包含 Donut 方法论对比及选择理由
